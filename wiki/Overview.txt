### Overview

__mkc__ is build configuration tool.  It will create a configuration
file based on the requested checks, generate makefiles for the project
(not yet implemented), build the project (not yet implemented), and
install the project (not yet implemented).

See [Status](Status) for the current status and goals.

##### Definitions

__profile__: A profile is a namespace for variables and attributes.
_mkc_ defines the __internal__ and __default__ profiles.

References: [Profiles](Profiles.txt)

A user profile can have any name.  The usual names might be
__development__, __debug__ and __release__.

A profile may have an associated compiler.

__attribute__: An attribute associated with a particular statement or
target.  An attribute might be a name, a set of compiler flags, etc.

__target__: An executable or library or other target that will be
built or processed.

Every target has its own profile for attributes and variables.

<br>References: [Syntax: Profile](SyntaxProfile.txt)

##### Running __mkc__

    mkc myproject.mkc

If some checks fail, and you want to re-check without removing the
cache, use the __&#8209;&#8209;retest__ command line argument.

    mkc --retest myproject.mkc

To select a different profile, use the __&#8209;&#8209;profile__
command line argument.

    mkc --profile debug myproject.mkc
