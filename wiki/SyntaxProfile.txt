### Syntax: profile

    profile <profile-name> { <statement-list> };

if _profile-name_ is active, then the statements in _statement-list_
will be run.  The profile is selected on the command line using the
__&#8209;&#8209;profile__ argument.

_profile-name_ is usually a user-defined name.
_mkc_ defined names are __internal__, __global__ and __release__.

When a variable is used, _mkc_ will check the profiles in the
following order:

* local variables (foreach, functions)
* target / compiler
* target
* user-profile / compiler
* user-profile
* global / compiler
* global
* internal
