### Status

##### 2026-6-24 pre-alpha

Basic syntax is working.  I believe I have figured a decent
syntax to handle the profiles/targets and their attributes.

Just working through everything, cleaning up, adding more checks.
The current goal is to get the configuration step all working,
and get all the infrastructure needed for a pre-release done.

The expected first pre-release will only create a configuration file.

##### Goals

* Goals
    * Straightforward, understandable algol-like syntax.
      Both _cmake_ and _mkconfig_ have weird syntax.
      Use a grammar so that the syntax is regular and extendable.
    * UTF-8 support.
    * Cross platform support.
    * Be able to associate checks with particular profiles/targets (complete).
    * Modern language support.
      Despite wanting to use _mkc_ for _di_ on legacy systems, I think
      it is better to just skip that and assume some modernity.
* Phase 1
    * Check for compiler flags. (partial)
    * Check for link flags. (partial)
    * Check for headers.
      Mostly not needed for modern systems.
      (not yet implemented)
    * Check for library functions. (c: complete)
    * Check for defines. (complete)
    * Check for types. (complete)
    * Check for structure members. (complete)
    * Check size of types. (complete)
    * Be able to name the checks. (complete)
    * Configure a file. (complete)
    * Very simple file support (variable=something).
      This is also useful for pkg-config files.
      (not yet implemented)
    * Package support (via pkg-config files).  (complete)
    * Variable cache. The usual invalidation problem.
      complete, but could use more tests for invalidation.
    * Test with the [libmp4tag](https://sourceforge.net/projects/libmp4tag/) package.
    * Test with _mkc_.
* Phase 2
    * Test with _di_.
      _di_ checks a __lot__ of stuff.  Many checks will be missing.
      Start adding these checks in.
    * Test with _mkc_.
    * Test with _libmp4tag_.
* Phase 3
    * Generate Makefiles
      This will be difficult.
    * Handle sub-directories
* Phase 4
    * Build executables.
    * Build shared libraries.
    * Test with _libmp4tag_.
    * Test with _di_.
    * Test with _mkc_.
    * At this point, an alpha release could be made.
* Phase 5
    * Installation process.
    * Test with _libmp4tag_.
    * Test with _di_.
    * Test with _mkc_.
    * alpha release.
* Phase 6
    * Test with _BallroomDJ 4_.
    * Test with _libmp4tag_.
    * Test with _di_.
    * Test with _mkc_.
    * beta release.
